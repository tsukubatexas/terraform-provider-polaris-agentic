# Code of Conduct

This project should be a calm, useful place to collaborate.

## Expected Behavior

- Be respectful and direct.
- Assume good intent, but correct unsafe or misleading work clearly.
- Keep technical disagreement focused on evidence, code, tests, and docs.
- Protect private information, credentials, customer data, and security findings.
- Make room for contributors with different experience levels.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing private messages, credentials, tokens, or personal information.
- Intentionally weakening security controls or misleading users about risk.
- Repeated low-effort disruption after maintainers ask for a change in behavior.

## Enforcement

Maintainers may edit or delete comments, close issues, block users, or report abuse to GitHub when needed to protect the project and its contributors.

Security issues should follow [SECURITY.md](SECURITY.md), not public issue threads.
